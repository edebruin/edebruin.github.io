Pokemon Mock Mini Game

The motivation for this project originated from our groups youth, growing up on the Pokemon games. We knew that a full replica would be a little too much to handle for a project that was only a few weeks long, but we thought it would be very fun to at least mock the sheer fundamentals of the game such as battling. It struck everyones interest because we knew what we were shooting for and it was really enjoyable to watch it take shape.

The project works by initializing you as a trainer and gives you a selection of starting pokemon. It then enters a loop asking the player a set of actions they can take. These range from catching a new pokemon, to battling a trainer, or swapping out pokemon with those in their PC. Each of these actions will take you to another function that will handle the users input. For our catching function we randomized which stage one pokemon appears. The user then has a roughly 80% chance to catch the specified pokemon. This pokemon would either be placed in the players party or ,if they had 6 pokemon in the party already, their PC. Each pokemon has base stat values linked as well as evolution level, and possible moves that can be learned. The main loop of actions will repeat until the user decides they are done playing the game and chooses to quit.

While the main focus for design was supposed to be test driven development, it was difficult to fully write our tests since the was much variance that could take place based on how far we wanted to take the project. Instead we wrote out test stubs for cases we knew we needed but left them blank to allow our project to be pliable. We broke the project down by having a group of classes for individual moves and another for the pokemon. Then we created a class to represent the user and a class to represent the NPC you would battle with. Finally we had our simulator class which is used to display our menus and run our program.

For this project I worked on the menus and formatting that the user has to interact with which is located in the simulator class. I also invested time into researching how base stats for each individual pokemon are used. For reference each pokemon has 6 stat values that are unique to its species. This stats are recalculated based on the level of the pokemon and if that pokemon evolves. Theses stats affect how much attack, hitpoints, defence, etc. a pokemon will have. I coded the equations for these stats, when they get recalculated and linked the unique set of numbers to all of the pokemon in our game.
